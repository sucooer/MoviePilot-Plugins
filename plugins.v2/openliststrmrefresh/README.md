# OpenList STRM 刷新

定时访问 OpenList STRM 驱动目录，触发 OpenList 生成本地 STRM 文件；也可以检测源目录变化，变化后自动刷新对应 STRM 目录。

## 使用前提

- MoviePilot 中已经添加名为 `alist` 的 OpenList 存储配置。
- OpenList 中已经配置 STRM 驱动。

## 功能

- 定时访问一个或多个 STRM 驱动目录。
- 为不同 STRM 目录配置不同刷新频率。
- 检测源目录变化，变化后刷新指定 STRM 目录。
- 支持递归访问子目录、递归深度限制和目录访问间隔。
- 支持常用频率、中文频率写法和自定义 cron。

## 配置

- `目录刷新计划`：无条件定时访问 STRM 目录，格式为 `频率 | 路径`。
- `变化检测计划`：检测源目录快照，变化后访问 STRM 目录，格式为 `频率 | 源目录 -> STRM目录`。
- `默认刷新频率` / `默认 STRM 驱动目录`：未填写计划时使用。
- `递归访问子目录` / `递归深度`：控制目录遍历范围。
- `强制刷新`：调用 OpenList 列目录接口时携带 `refresh=true`。

## 示例

不同目录使用不同刷新频率：

```text
每6小时 | /strm/电影
每天03点 | /strm/剧集,/strm/动漫
```

检测源目录变化后刷新 STRM 目录：

```text
每15分钟 | /原驱动/电影 -> /strm/电影
每30分钟 | /原驱动/剧集 -> /strm/剧集
```
