# OpenList STRM 刷新

定时访问 OpenList STRM 驱动目录，触发 OpenList 生成本地 STRM 文件。

## 使用前提

- MoviePilot 中已经添加名为 `alist` 的 OpenList 存储配置。
- OpenList 中已经配置 STRM 驱动。

## 配置说明

- `目录刷新计划`：一行一个计划，格式为 `频率 | 路径`。多个路径可用英文逗号分隔。
- `默认刷新频率`：当 `目录刷新计划` 为空时使用，可直接选择常用频率。
- `默认 STRM 驱动目录`：当 `目录刷新计划` 为空时使用。一行一个路径，例如 `/strm`。
- `自定义 cron`：只有 `默认刷新频率` 选择自定义 cron 时使用。
- `递归访问子目录`：开启后会继续访问子目录，用于触发更多层级的 STRM 生成。
- `递归深度`：限制递归层级，避免目录过深时任务运行过久。
- `强制刷新`：调用 OpenList 列目录接口时携带 `refresh=true`。

不同目录使用不同频率的示例：

```text
每6小时 | /strm/电影
每天03点 | /strm/剧集,/strm/动漫
```

`目录刷新计划` 左侧既可以写常用中文频率，也可以直接写 5 位 cron。

## 工作方式

插件会调用 OpenList 的 `/api/fs/list` 接口访问配置目录。OpenList STRM 驱动在目录被访问时生成本地 STRM 文件，因此定时访问目录即可补足 OpenList 当前没有自动定时触发的问题。
